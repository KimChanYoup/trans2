## 문제 1 — localStorage 헬퍼 함수 중복 ✅ 수정 완료

**원인:** 각 페이지 파일 상단에 localStorage read/write 함수를 직접 정의하여 동일한 함수가 5개 파일에 복붙되어 있었음.

| 중복 함수명 | 발견된 파일 | 비고 |
|------------|-----------|------|
| `loadOwnedHeroes()` / `saveOwnedHeroes()` | `GamePage.tsx`, `OffenseGamePage.tsx`, `HeroesPage.tsx` | 3중 중복 |
| `loadRaidStageProgress()` / `saveRaidStageProgress()` | `GamePage.tsx`, `AIGamePage.tsx` | 2중 중복 |
| `loadWallTalents()` / `saveWallTalents()` | `GamePage.tsx`, `AIGamePage.tsx` | 2중 중복 |
| `loadStageProgress()` / `saveStageProgress()` | `OffenseGamePage.tsx`, `HeroesPage.tsx` | 2중 중복 |
| `loadProtagonistSave()` 등 13개 | `HeroesPage.tsx` → 다른 파일들이 import | 출처 파일 부적절 |
| `loadManghonguSave()` / `saveManghonguData()` | `game/manghonguData.ts` | 분리 불일치 |

**수정:** `frontend/src/utils/localStorage.ts` 단일 파일로 통합, 중복 함수 18개 제거

## 문제 2 — 백엔드 `req: any` 패턴 ✅ 수정 완료

**원인:** NestJS에서 JWT 인증된 `req.user`를 접근할 때 모든 컨트롤러에서 `req: any` 사용.

**수정:** `backend/src/types/auth.types.ts` 신규 생성 후 7개 컨트롤러 32건 교체

```typescript
// 수정 전
async logout(@Request() req: any) { return this.authService.logout(req.user.id); }

// 수정 후
async logout(@Request() req: AuthenticatedRequest) { return this.authService.logout(req.user.id); }
```

**영향 파일:** `auth`, `user`, `achievement`, `leaderboard`, `tournament`, `friend`, `shop` 컨트롤러

## 문제 3 — 프론트엔드 `any` 타입 남용 (총 180건)

### 1차 수정 완료 ✅ (16건 제거, 단순 패턴)

| 파일 | 제거 건수 | 수정 내용 |
|------|----------:|---------|
| `LoginPage.tsx` | 2 | `catch (err: any)` 제거, `(t.login as any)` → `t.login` |
| `RegisterPage.tsx` | 4 | `catch (err: any)` → `catch (err: unknown)`, 번역 캐스팅 제거 |
| `FriendsPage.tsx` | 6 | `(t as any).friends` → `t.friends`, `catch (err: any)` × 4 |
| `ProfilePage.tsx` | 1 | `catch (err: any)` → `catch (err: unknown)` |
| `Layout.tsx` | 3 | `res.data as any[]` 제거, 번역 캐스팅 제거, 타입 인터페이스 보완 |

**추가 생성:** `frontend/src/utils/error.ts` — API 에러 메시지 안전 추출 헬퍼

### 2차 수정 필요 ⏳ (게임 로직 관련 복잡한 패턴)

## 문제 6 — 에러 처리 패턴 불일치 ⏳ 미완료

### 백엔드 — try/catch 없는 서비스 파일

| 파일 | 문제 |
|------|------|
| `user.service.ts` | DB 쿼리 try/catch 없이 직접 throw에 의존 |
| `shop.service.ts` | Exception 던지기만 있고 처리 없음 |
| `friend.service.ts` | 에러 처리 미흡 |
| `tournament.service.ts` | in-memory 조작 시 에러 처리 없음 |

### 프론트엔드 — try/catch 있는 것/없는 것 혼재
- API 호출 절반 이상에 try/catch 없음
- 에러 메시지 표시 방식도 파일마다 다름
