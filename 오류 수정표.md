## #1 — Footer가 화면 중간에 뜨는 문제

**파일:** `frontend/src/components/Layout.tsx`  
**발견 경위:** F12 반응형 화면 테스트 시, 콘텐츠가 짧은 페이지에서 footer가 하단이 아닌 중간에 위치함

### 원인

`mt-auto`는 부모가 `flex` 컨테이너일 때만 동작한다.  
현재 최상위 `<div>`는 `min-h-screen`만 있고 `flex flex-col`이 없어서 `mt-auto`가 무시됨.

```jsx
// Layout.tsx:108 — 현재 (문제 있음)
<div className="min-h-screen bg-gray-900 text-white">
  <nav>...</nav>
  <main><Outlet /></main>
  <footer className="... mt-auto">...</footer>  // ← mt-auto 무효
</div>
```

### 수정 내용

| 위치 | 기존 코드 | 수정 코드 |
|------|-----------|-----------|
| `Layout.tsx:108` 최상위 div | `min-h-screen bg-gray-900 text-white` | `min-h-screen bg-gray-900 text-white flex flex-col` |
| `Layout.tsx:244` `<main>` | `<main>` | `<main className="flex-1">` |

### 수정 후 동작

- 콘텐츠가 짧아도 footer가 항상 화면 맨 아래에 고정됨
- 콘텐츠가 길면 자연스럽게 아래로 밀림

## #3 — Backend 포트 3000 외부 노출 (HTTPS 우회)

**파일:** `docker-compose.yml`  
**발견 경위:** 디펜스 평가 중 평가자 지적 — "포트 3000이 열려 있으면 뚫린 것 아니냐"

### 원인

`ports`는 컨테이너 포트를 호스트 외부로 직접 노출하는 설정이다.  
backend와 db 모두 `ports`가 설정되어 있어 Nginx를 완전히 우회한 접근이 가능했다.

```
의도한 경로:   외부 → Nginx(HTTPS) → backend (암호화 O)
실제 가능했던 경로: 외부 → http://서버:3000 → backend (암호화 X, HTTPS 우회)
                  외부 → 서버:5432 → PostgreSQL 직접 접근 (매우 위험)
```

과제 요구사항 위반: *"For the backend, HTTPS must be used everywhere."*

### 수정 내용

| 서비스 | 기존 | 수정 |
|--------|------|------|
| `backend` | `ports: - "${BACKEND_PORT:-3000}:3000"` | `ports` 항목 제거 |
| `db` | `ports: - "${DB_PORT:-5432}:5432"` | `ports` 항목 제거 |

### 수정 후 동작

- backend, db 모두 `app-network` 내부에서만 접근 가능
- 외부에서 접근 가능한 포트는 Nginx의 8080(HTTP→HTTPS 리다이렉트), 8443(HTTPS)만 존재
- 모든 외부 통신이 반드시 Nginx를 거쳐 HTTPS로 암호화됨

## #4 — HTTP → HTTPS 리다이렉트 포트 오류

**파일:** `nginx/nginx.conf`  
**발견 경위:** 디펜스 평가 "안전한 연결" 항목 X — HTTP 접속 시 HTTPS로 정상 전환되지 않음

### 원인

`nginx.conf:15`의 HTTP → HTTPS 리다이렉트에서 `$host` 변수를 사용했는데,  
nginx의 `$host`는 **포트 번호를 포함하지 않는다.**

```
사용자 접속: http://localhost:8080/
$host       = "localhost"  (포트 없음)
리다이렉트  → https://localhost/  (443 포트 시도)
실제 HTTPS  = https://localhost:8443/
결과: 연결 실패 — 포트가 달라서 브라우저가 접속 못함
```

### 수정 내용

| 파일 | 기존 코드 | 수정 코드 |
|------|-----------|-----------|
| `nginx/nginx.conf:15` | `return 301 https://$host$request_uri;` | `return 301 https://$host:8443$request_uri;` |

### 수정 후 동작

```
사용자 접속: http://localhost:8080/
리다이렉트  → https://localhost:8443/  ← 정확한 포트로 이동
```

### 참고 — 자체 서명 인증서 경고

`make` 실행 시 `openssl`로 자체 서명 인증서(`cert.pem`, `key.pem`)를 자동 생성한다.  
브라우저에서 "안전하지 않음" 경고가 뜨는 것은 인증 기관(CA)의 서명이 없기 때문이며,  
**HTTPS 암호화 자체는 정상 동작**한다. "고급 → 계속"으로 접속하면 된다.  
(Makefile에도 이 안내가 명시되어 있음: `⚠ 처음 접속 시 '안전하지 않음' 경고 → 고급 → 계속`)

## #5 — 로비 3인 플레이 시 게스트 패배 화면에 나가기 버튼 2개 중복

**파일:** `frontend/src/pages/GamePage.tsx`

### 원인

게스트 패배 시 두 조건이 동시에 참이 되어 두 블록이 모두 렌더링됨.

- 블록 A (line 2041): `isGuest && guestGameEnded` → 게스트 전용 종료 UI (메인화면 버튼 포함)
- 블록 B (line 2057): `gameState.phase === 'defeat'` → 일반 Game Over Actions (메인화면 버튼 포함)

`handleGameEnded` 핸들러가 `setGameState({ phase: 'defeat' })`와 `setGuestGameEnded(result)`를 동시에 호출하므로 두 조건이 모두 참이 됨. 블록 B에 `!isGuest` 가드가 없었음.

### 수정 내용

| 파일 | 위치 | 기존 코드 | 수정 코드 |
|------|------|-----------|-----------|
| `GamePage.tsx` | line 2057 | `gameState.phase === 'victory' \|\| gameState.phase === 'defeat') && (` | `gameState.phase === 'victory' \|\| gameState.phase === 'defeat') && !isGuest && (` |

### 수정 후 동작

- 게스트: 블록 A(게스트 전용 UI)만 표시 — 나가기 버튼 1개
- 호스트/솔로: 블록 B(일반 Game Over Actions)만 표시

### 타 페이지 동일 버그 여부

## #7 — LAN(동일 네트워크) 다중 컴퓨터 멀티플레이 불가 (CORS 제한)

**파일:** `backend/src/main.ts`, `backend/src/game/game.gateway.ts`  
**발견 경위:** 42 클러스터 환경에서 서로 다른 컴퓨터로 로비 멀티플레이를 시도 시 WebSocket 연결 실패 및 API 요청 차단

### 원인

REST API CORS(`main.ts`)와 WebSocket CORS(`game.gateway.ts`) 모두 `localhost`만 허용하는 정규식을 사용:

```typescript
/^https?:\/\/localhost(:\d+)?$/
```

다른 컴퓨터에서 서버 PC의 IP(예: `192.168.x.x`)로 접속하면:
- HTTP API 요청 → 백엔드에서 CORS 오류로 차단
- WebSocket(`/ws`) 연결 → Gateway에서 CORS 오류로 차단
- 결과: 로그인, 로비 입장, 게임 소켓 모두 실패

### 수정 내용

두 파일 모두 동일하게 사설망(RFC 1918) IP 대역을 허용 범위에 추가:

| 허용 대역 | 범위 | 용도 |
|-----------|------|------|
| `10.0.0.0/8` | `10.x.x.x` | 클래스 A 사설망 (42 클러스터 등) |
| `172.16.0.0/12` | `172.16–31.x.x` | 클래스 B 사설망 |
| `192.168.0.0/16` | `192.168.x.x` | 클래스 C 사설망 (가정/사무실 공유기) |

```typescript
// 수정 후 — main.ts & game.gateway.ts 동일 적용
if (
  !origin ||
  /^https?:\/\/localhost(:\d+)?$/.test(origin) ||
  /^https?:\/\/(10\.\d+\.\d+\.\d+|172\.(1[6-9]|2\d|3[01])\.\d+\.\d+|192\.168\.\d+\.\d+)(:\d+)?$/.test(origin)
) {
  callback(null, true);
}
```

### 수정 후 동작

- 서버 PC(Docker 실행) IP를 아는 동일 네트워크의 모든 컴퓨터가 접속 가능
- 2인 파티 및 3인 레이드 멀티플레이 모두 작동
- 인터넷(다른 네트워크)에서의 접근은 여전히 차단 (보안 유지)
