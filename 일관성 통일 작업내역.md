## 작업 1 — 백엔드 공통 인증 타입 생성

### 신규 파일 생성
**`backend/src/types/auth.types.ts`** (새 파일)

```typescript
export interface AuthUser {
  id: number;
  email: string;
  username: string;
}

export interface AuthenticatedRequest {
  user: AuthUser;
}
```

### 문제
- 거의 모든 컨트롤러에서 `@Request() req: any`를 사용
- `req.user.id`, `req.user.username` 접근 시 타입 안전성 없음
- JWT 인증된 요청에 대한 타입 정의가 전혀 없었음

### 수정 내용
아래 7개 컨트롤러 전체에서 `req: any` → `req: AuthenticatedRequest` 교체:

| 파일 | 교체 건수 |
|------|----------:|
| `backend/src/auth/auth.controller.ts` | 1 |
| `backend/src/user/user.controller.ts` | 13 |
| `backend/src/achievement/achievement.controller.ts` | 2 |
| `backend/src/leaderboard/leaderboard.controller.ts` | 1 |
| `backend/src/tournament/tournament.controller.ts` | 5 |
| `backend/src/friend/friend.controller.ts` | 8 |
| `backend/src/shop/shop.controller.ts` | 2 |
| **합계** | **32** |

추가로 `user.controller.ts`의 `saveData: any` 2건도 `saveData: unknown`으로 교체.

## 작업 4 — 프론트엔드 `any` 타입 제거 (1차)

### 신규 파일 생성
**`frontend/src/utils/error.ts`** — API 에러 메시지 안전 추출 헬퍼

```typescript
export function getErrorMessage(err: unknown, fallback = '오류가 발생했습니다.'): string {
  if (err && typeof err === 'object') {
    const responseMsg = (err as { response?: { data?: { message?: string } } })
      ?.response?.data?.message;
    if (typeof responseMsg === 'string') return responseMsg;
    if (typeof (err as { message?: string }).message === 'string')
      return (err as { message: string }).message;
  }
  return fallback;
}
```

### 수정된 파일 (any 0건으로 감소)

| 파일 | 수정 내용 | 제거 건수 |
|------|-----------|----------:|
| `LoginPage.tsx` | `catch (err: any)` → `catch`, `(t.login as any)` → `t.login` | 2 |
| `RegisterPage.tsx` | `catch (err: any)` → `catch (err: unknown)`, `(t.register as any)` → `t.register` | 4 |
| `FriendsPage.tsx` | `(t as any).friends` → `t.friends`, `(t as any).common` → `t.common`, `catch (err: any)` × 4 | 6 |
| `ProfilePage.tsx` | `catch (err: any)` → `catch (err: unknown)` | 1 |
| `Layout.tsx` | `res.data as any[]` → `res.data as Notification[]`, `(t as any).notification` → `t.notification`, `Notification` 인터페이스에 누락된 `unlocked` 필드 추가 | 3 |
| **합계** | | **16** |

### 수정 패턴

**패턴 1 — `catch (err: any)` 제거:**
```typescript
// 수정 전
} catch (err: any) {
  setError(err.response?.data?.message || '오류');
}

// 수정 후
} catch (err: unknown) {
  const msg = (err as { response?: { data?: { message?: string } } })?.response?.data?.message;
  setError(msg || '오류');
}
```

**패턴 2 — `(t as any).xxx` 제거:**
```typescript
// 수정 전 — 번역 키가 실제로 존재하는데 불필요하게 any 캐스팅
(t.login as any).loginFailed
(t as any).notification?.title

// 수정 후 — 직접 접근
t.login.loginFailed
t.notification?.title
```
